What does the -l flag to ls do? Run ls -l / and examine the output.
What do the first 10 characters of each line mean? (Hint: man ls)

answer:
the 'l' flag will make the result of ls become verbose, the first 10
lines indicate the file type and permissions, (-) for file and ( d )
for directory, followed by read, write execute permisions.

some files and experiment with patterns like ls *.txt, ls file?.txt,
and ls {a,b,c}.txt. See Pattern Matching in the Bash manual.

answer:
glob mean to match none or any characters at that exact position,
meaning the "*.zip" will match any file that start with any character
and will end with ".zip"
